## 2024-05-04 - React.memo Optimization Learnings
**Learning:** Adding `React.memo` is only half the optimization; passing inline arrow functions (like `onOpenComments={(id) => onOpenComments(id)}`) to a child component defeats the purpose of memoizing that child since the function reference changes on every render.
**Action:** Always inspect the props being passed to a component wrapped in `React.memo`. If an inline arrow function is not necessary, pass the function reference directly (e.g., `onOpenComments={onOpenComments}`) to preserve referential equality and ensure the `React.memo` optimization is effective. Additionally, wrap components using `export const ComponentName = memo(function ComponentName(...)` to maintain named exports and component names in React DevTools.
