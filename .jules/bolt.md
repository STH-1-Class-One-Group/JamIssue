## 2024-05-23 - Optimize single-item and 1-to-many array state updates
**Learning:** In React, unconditional `.map()` calls within state updaters for single items or 1-to-many relationships (like updating comments for a specific review) cause O(N) memory allocations and break referential equality even when no items match. The codebase previously relied on `.map()` heavily in `src/hooks/` (e.g. `useAppReviewCrudActions`, `useAppAdminActions`).
**Action:** Replace single-item updates with `.findIndex()`. If the index is not found, return the original array reference. If found, shallow-copy the array and update only the matched index. For 1-to-many updates (like comments), use a single `for` loop that defers array cloning until the first matching item is found, preventing O(N) allocations when no updates are needed.
