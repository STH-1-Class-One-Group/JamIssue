## 2026-05-26 - Eliminate O(N) intermediate array allocations in `.filter().length` count calculations
**Learning:** In state stores and logic files (like `notification-store` and `visits.ts`), counting items meeting a condition via `array.filter(condition).length` causes unnecessary intermediate array creation. This results in $O(N)$ memory overhead and triggers rapid GC cycles when running on high volume data arrays. The pattern occurred heavily during frequent state updates like realtime notification processing and place count renders.
**Action:** Always refactor `.filter(condition).length` into a simple `for...of` loop with an internal counter variable to achieve identical results with $O(1)$ memory complexity, preventing runtime bloat across React re-renders and Zustand actions.
