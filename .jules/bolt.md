## 2024-05-24 - Prevent GC Pressure in React Effects
**Learning:** When deriving multiple state structures (like a Set of IDs, a Map for lookups, and a string signature) from the same source array in a frequently-running React effect (such as Naver Map panning), chaining or running multiple `.map()` passes creates unnecessary intermediate array allocations that spike garbage collection pressure. Additionally, using the spread operator to combine mapped arrays (`[...arr1.map(fn1), ...arr2.map(fn2)]`) causes similar O(N) array allocation overhead.
**Action:** Consolidate multiple iteration passes into a single `for...of` loop to build multiple structures simultaneously. When constructing arrays of operations or batched closures, initialize an empty array (`const operations = []`) and use `.push()` inside sequential `for...of` loops over the constituent arrays to eliminate intermediate array creation.
