## 2026-06-21 - [Optimize chained map and filter with nested loop]
**Learning:** Chaining `.map().filter()` on array structures, especially inside React render cycles (like in `TourismInfoSheet`), causes unnecessary intermediate array allocations, increasing GC pressure and degrading performance for high-frequency renders. This is especially true if the `map` function applies expensive string transformations before checking if the result should be filtered.
**Action:** Replace  chains with a single  loop. Only apply expensive operations (like string translation/normalization) immediately before checking for validity, and conditionally push to a single resulting array to save memory allocation and CPU cycles.
