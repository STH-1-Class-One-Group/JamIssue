## 2025-05-05 - Avoid list-wide renders during item-level edits
**Learning:** In a list of cards where a user can toggle an "edit mode" for a single item, keeping the volatile form state (e.g., input values) in a parent hook (`useMyFeedReviewEditor`) and blindly spreading it to all cards breaks `React.memo()`. As the user types, all cards in the list re-render on every keystroke because the volatile prop values change.
**Action:** When hoisting edit state for a list, use `React.memo()` on the list item component, ensure callback functions are stable (e.g. `useEventCallback`), and conditionally pass the rapidly changing state props ONLY to the item being actively edited. Pass stable default primitives (`''`, `false`, `null`) to the other inactive items to preserve their referential equality.
