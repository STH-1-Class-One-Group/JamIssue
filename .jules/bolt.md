## 2026-05-08 - [O(N) Re-render in list components due to primitive string IDs]
**Learning:** Passing a globally changing primitive string ID (like `likingReviewId`) down to every memoized list item breaks `React.memo()`. Because the prop changes for *every* item (e.g., from `null` to `some-id`), it triggers an O(N) re-render of the entire feed. This is a severe performance bottleneck for long infinite-scroll feeds. The "null 덮어쓰기" (null overriding) approach was rejected by the project previously, so that cannot be used.
**Action:** When a specific item in a list needs to react to a global state (like being liked, highlighted, or submitted), compute a boolean flag derived from the ID *at the list level* (e.g., `isLiking={likingReviewId === review.id}`) and pass the boolean to the memoized list item. This ensures only the items whose boolean state actually changes will re-render, reducing O(N) complexity to O(1).
