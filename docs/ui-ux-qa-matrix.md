# UI/UX 기대 동작 QA 매트릭스

## 목적

이 문서는 화면에서 기대한 것과 다르게 동작하는 항목을 추적하기 위한 기준 문서다. UI/UX 버그는 `UIUX-###` 기대 동작 ID, 재현 경로, 자동 테스트 여부, 수동 QA 여부, 관련 이슈와 PR을 함께 기록한다.

## 운영 규칙

| 항목 | 기준 |
| --- | --- |
| ID 형식 | `UIUX-###` |
| 단위 | 사용자가 인지하는 화면 동작 하나 |
| 자동화 기준 | unit, integration, regression, E2E 중 하나로 재현 가능하면 자동 테스트 연결 |
| 수동 QA 기준 | 실제 iPhone Safari/WebView처럼 자동화가 불완전한 항목은 수동 근거 기록 |
| 이슈 작성 기준 | 기대 동작 ID, 실제 동작, 재현 경로, 테스트 상태, 관련 PR을 포함 |
| 완료 기준 | 자동 테스트 또는 수동 QA 근거 없이 close 금지 |

## 기대 동작 ID

| ID | 영역 | 기대 동작 | 재현 경로 | 자동화 상태 | 수동 QA |
| --- | --- | --- | --- | --- | --- |
| `UIUX-001` | 전역 레이아웃 | 모바일 기준 폭에서 하단 탭은 화면 하단에 고정되고 주요 패널과 겹치지 않는다. | 앱 진입 -> 탭 전환 | 예정: Playwright | 필요 |
| `UIUX-002` | 탭 전환 | `지도` 외 탭에서는 지도 배경이 보이지 않고 각 탭이 독립 화면처럼 보인다. | 지도 -> 피드/코스/마이 | 예정: Playwright | 선택 |
| `UIUX-003` | 지도 레이어 | 상단 카드, 필터, 지도, 드로워, 알림/피드백 진입점, Naver attribution이 서로 잘못 올라오지 않는다. | 지도 -> 알림/피드백 열기 | 예정: Playwright + 수동 | 필요 |
| `UIUX-004` | 장소 드로워 | 장소 마커 선택 시 드로워가 `partial`, `full`, `closed` 상태로 자연스럽게 전환된다. | 지도 -> 장소 선택 -> 드래그 | 예정: Playwright | 필요 |
| `UIUX-005` | 장소 정보 | 장소 드로워는 장소명, 구, 요약, 카테고리, 방문/스탬프 상태, 후기 작성 가능 여부, 후기 목록을 보여준다. | 지도 -> 장소 선택 | 예정: integration/E2E | 선택 |
| `UIUX-006` | 축제 정보 | 축제 드로워는 정보 제공만 담당하고 스탬프 로직과 섞이지 않는다. | 지도 -> 행사 선택 | 예정: E2E | 선택 |
| `UIUX-007` | 스탬프 버튼 | 로그아웃, 반경 밖, 반경 안, 오늘 완료 상태가 각각 기대 상태로 표시된다. | 지도 -> 장소 선택 -> 스탬프 버튼 | 예정: E2E + service test | 필요 |
| `UIUX-008` | 후기 작성 진입 | 후기는 당일 해당 장소 스탬프가 있을 때만 열리고, GPS 진입만으로는 열리지 않는다. | 스탬프 전/후 -> 후기 작성 | 예정: E2E | 필요 |
| `UIUX-009` | 모바일 키보드 | 리뷰 작성 textarea focus 시 키보드가 떠도 하단 탭과 드로워가 화면 중간으로 떠오르지 않는다. | 지도 -> 장소 -> 스탬프 완료 -> 후기 textarea focus | unit/integration 있음, E2E 예정 | 필수 |
| `UIUX-010` | 피드 탭 | 피드 탭은 실제 방문 후기 소비 중심이며 댓글, 좋아요, 장소 이동 CTA가 동작한다. | 피드 -> 댓글/좋아요/장소 이동 | 예정: E2E | 선택 |
| `UIUX-011` | 코스 탭 | 코스 탭은 `좋아요순 / 최신순` 정렬과 카드 진입을 제공한다. | 코스 -> 정렬 -> 카드 선택 | 예정: E2E | 선택 |
| `UIUX-012` | 마이 탭 | 로그인 전에는 로그인 유도만, 로그인 후에는 개인 통계와 내부 탭이 표시된다. | 마이 -> 로그인 전/후 | 예정: E2E | 선택 |
| `UIUX-013` | 마이 피드 수정 | 내 피드 수정 진입 시 inactive card가 불필요하게 재렌더되지 않고 편집 UI가 안정적으로 동작한다. | 마이 -> 내가 쓴 피드 -> 수정 | unit 있음, E2E 예정 | 선택 |
| `UIUX-014` | 알림/피드백 진입 | 알림과 피드백은 설정 진입점 아래에서 접근되고 지도 레이어를 깨지 않는다. | 우측 상단 설정 -> 알림/피드백 | 예정: E2E | 필요 |

## 이슈 템플릿

```markdown
## 기대 동작 ID

- UIUX-###

## 기대 동작

-

## 실제 동작

-

## 재현 경로

1. 

## 테스트 상태

- 자동 테스트:
- 수동 QA:

## 관련 근거

- Issue:
- PR:
- Merge SHA:
- CI:
```

## PR 체크 기준

- UI/UX 관련 PR은 영향받는 `UIUX-###` ID를 PR 본문에 기록한다.
- 자동 테스트가 있으면 테스트 파일과 명령을 기록한다.
- 자동화가 불완전한 항목은 수동 기기, 브라우저, 일시, 확인자를 기록한다.
- 기대 동작이 새로 생기면 이 문서에 ID를 먼저 추가한다.
