# JamIssue 기술 상세 명세서 및 발표 자료 (Technical Overview)

이 문서는 대전 로컬 장소·행사·리뷰 탐색 서비스 **JamIssue**의 기술적인 구현 내용과 핵심 기능을 PPT 제작 및 발표 대본용으로 상세히 정리한 문서입니다.

---

## 1. 서비스 개요 (Service Overview)

### 1.1 서비스의 목적
- **지역 활성화**: 대전 시내의 카페, 식당, 문화공간 등 로컬 장소의 탐색을 돕고 방문을 유도합니다.
- **경험의 기록**: 단순 검색을 넘어 방문 인증(스탬프), 리뷰(피드), 경로 생성(코스)으로 이어지는 사용자 경험을 제공합니다.
- **정보의 통합**: 대전시 공식 행사 일정과 로컬 장소 정보를 지도를 중심으로 한곳에 모아 제공합니다.

### 1.2 서비스의 가치 (Value Proposition)
- "노잼 도시 대전을 유잼 도시로" - 대전의 숨겨진 장소들을 발견하는 즐거움을 제공합니다.
- **신뢰성 있는 후기**: 실제 위치 인증을 거친 사용자만 후기를 작성할 수 있도록 하여 데이터의 품질을 높입니다.
- **커뮤니티 기반 코스**: 운영자가 일방적으로 제공하는 코스가 아닌, 사용자가 직접 걸어본 실제 동선을 공유합니다.

## 2. 핵심 기능 상세 (Core Features)

### 2.1 지도 기반 탐색 (Map-Centric Discovery)
- **네이버 지도 API 연동**: 대전 전역의 랜드마크, 맛집, 카페 정보를 카테고리별로 필터링하여 확인 가능합니다.
- **바텀 드로워(Bottom Drawer) UX**: 지도에서 장소/행사 마커 클릭 시 하단에서 상세 정보가 슬라이드업되는 직관적인 UI를 제공합니다. (`closed / partial / full` 3단계 상태 관리)

### 2.2 위치 인증 기반 스탬프 시스템 (Location-based Stamp)
- **반경 기반 검증**: 사용자의 현재 GPS 좌표와 장소의 좌표를 비교하여, 설정된 반경(**120m**) 이내에 있을 때만 스탬프를 찍을 수 있습니다.
- **방문 횟수 추적**: `visit_ordinal` 필드를 통해 동일 장소의 다회차 방문(n번째 방문)을 기록하고 시각화합니다.

### 2.3 방문 증명형 리뷰 및 소셜 피드 (Proof-of-Visit Feed)
- **후기 작성 제한**: 당일 해당 장소의 스탬프를 찍은 사용자만 리뷰 작성이 가능합니다. (`stamp_id` 기반 검증)
- **소셜 인터랙션**: 리뷰에 대한 좋아요, 댓글, 대댓글(깊이 1단계) 기능을 제공합니다.

### 2.4 여행 세션 및 커뮤니티 경로 (Travel Session & User Routes)
- **24시간 세션 규칙**: 사용자가 스탬프를 찍으면 자동으로 세션이 생성되거나 기존 세션에 포함됩니다. 마지막 스탬프 후 24시간 이내의 활동을 하나의 '여행 세션'으로 묶습니다.
- **코스 발행**: 한 세션에서 2곳 이상의 장소를 방문했을 경우, 이를 '커뮤니티 경로'로 발행하여 다른 사용자와 공유할 수 있습니다.

### 2.5 실시간 알림 체계 (Notification System)
- **SSE(Server-Sent Events) 아키텍처**: 댓글, 좋아요 발생 시 페이지 새로고침 없이 실시간으로 알림 배지가 갱신됩니다.
- **인메모리 브로커**: Worker 환경에서 실시간 이벤트를 중개하여 사용자별 알림 스트림을 제공합니다.

### 2.6 공공 데이터 연동 (Public Data Integration)
- **주간 행사 동기화**: 대전시 공식 행사 상세검색 페이지를 매주 월요일 03:00 KST에 크롤링하여 DB에 업데이트합니다. (GitHub Actions 사용)

## 3. 기술 아키텍처 (Technical Architecture)

### 3.1 하이브리드 백엔드 전략 (Worker-First)
- **Cloudflare Workers**: 공개 API의 단일 진입점 역할을 하며, 인증, 리뷰, 스탬프 등 핵심 비즈니스 로직을 직접 처리하여 빠른 응답 속도를 보장합니다.
- **FastAPI (Origin)**: 로컬 개발 환경 및 Worker가 처리하지 않은 복잡한 백오피스/관리자 기능을 보조하는 용도로 활용됩니다.

### 3.2 프론트엔드 구성
- **React + TypeScript**: 안정적인 타입 시스템 기반의 컴포넌트 개발.
- **Zustand**: 가볍고 직관적인 전역 상태 관리를 통해 복잡한 지도 및 사용자 데이터 상태를 효율적으로 관리합니다.
- **Vite & Esbuild**: 초고속 빌드 및 번들링.

### 3.3 인프라 및 스토리지
- **Supabase**: PostgreSQL 기반의 운영 DB, 이미지 업로드를 위한 Storage, Realtime 기능을 활용합니다.
- **Cloudflare Pages**: 프론트엔드 정적 자산 배포 및 런타임 환경 주입.

## 4. 데이터 모델링 (Data Modeling)

### 4.1 주요 테이블 구조
- `user` / `user_identity`: 통합 계정 체계 및 소셜 로그인(네이버) 정보.
- `map`: 장소 마스터 데이터 (좌표, 카테고리, 스타일 테마 등).
- `user_stamp`: 사용자별 장소 방문 기록 및 증명.
- `travel_session`: 스탬프 기록을 시간 단위로 묶어주는 세션 엔티티.
- `user_route` / `user_route_place`: 세션 기반으로 생성된 사용자 코스.
- `feed` / `feed_like` / `user_comment`: 소셜 활동 데이터.
- `user_notification`: 실시간 알림 기록.

### 4.2 핵심 비즈니스 로직 (SQL/Logic)
- **정합성 검증**: 리뷰 생성 시 `Feed.stamp_id`가 실제 사용자의 소유인지, 해당 장소와 일치하는지 API 레벨에서 엄격하게 검증합니다.
- **세션 자동화**: 스탬프 생성 시 `repository._find_or_create_travel_session` 로직을 통해 24시간 이내의 기록을 자동으로 체인화합니다.

## 5. UI/UX 디자인 가이드라인 (Design Spec)

### 5.1 전역 규칙
- **Mobile First**: 390px 기준의 모바일 뷰 최적화.
- **색상 체계**: 장소 카테고리에 따른 동적 컬러링 (`jam_color`, `accent_color`)을 사용하여 지도 상의 시각적 구분을 명확히 합니다.

### 5.2 네비게이션 구조 (4 Tabs)
1. **지도**: 장소 및 축제 탐색의 시작점.
2. **피드**: 다른 사용자들의 실제 방문 후기 소비 공간.
3. **코스**: 사용자 및 운영자 큐레이션 경로 탐색.
4. **마이**: 본인의 활동(스탬프, 후기, 경로) 및 알림 관리.

## 6. PPT 발표용 대본 및 핵심 포인트 (Presentation Scripts)

### [슬라이드 1: 서비스 소개]
- **대본**: "안녕하세요. 대전의 숨은 매력을 발견하고 기록하는 서비스, JamIssue 발표를 시작합니다. 저희는 '노잼'이라 불리는 대전의 이미지를 타파하고, 로컬 장소를 탐색하며 실제 방문 기록을 남길 수 있는 플랫폼을 지향합니다."
- **포인트**: 대전 지역 특화, 서비스의 밝고 활기찬 이미지 강조.

### [슬라이드 2: 핵심 기술 - 위치 인증]
- **대본**: "JamIssue의 핵심은 '신뢰'입니다. 저희는 사용자가 실제 그 장소에 있을 때만 스탬프를 찍을 수 있도록 120m 반경의 GPS 검증 로직을 구현했습니다. 이 스탬프는 리뷰 작성의 필수 조건이 되어, 거짓 없는 진짜 후기 생태계를 만듭니다."
- **포인트**: 120m 반경 검증 로직, `stamp_id` 기반 리뷰 작성 제한.

### [슬라이드 3: 아키텍처 - Cloudflare & Supabase]
- **대본**: "기술적으로는 Cloudflare Workers와 Supabase를 활용한 하이브리드 아키텍처를 채택했습니다. API 응답 속도를 최적화하기 위해 Worker에서 핵심 로직을 처리하며, SSE를 통해 댓글이나 좋아요 알림을 실시간으로 전달합니다."
- **포인트**: 서버리스 아키텍처의 속도와 효율성, SSE 실시간성.

### [슬라이드 4: 사용자 여정 - 여행 세션]
- **대본**: "사용자의 활동은 단순히 점으로 남지 않습니다. 24시간 이내의 활동을 하나의 '세션'으로 묶어, 사용자가 대전에서 보낸 하루의 동선을 자동으로 기록합니다. 이렇게 생성된 동선은 클릭 몇 번으로 멋진 코스가 되어 다른 여행자들에게 공유됩니다."
- **포인트**: 24시간 세션 규칙, 개별 스탬프의 코스화(Chain-up).

## 7. 기술적 한계 및 향후 과제
- **인메모리 브로커 확장**: 현재 SSE 알림 브로커는 인메모리 방식이나, 향후 수평 확장을 위해 Redis Pub/Sub 도입을 검토 중입니다.
- **관리자 기능 고도화**: 운영 효율성을 위한 관리자 대시보드 및 콘텐츠 모더레이션 기능 추가 예정입니다.
- **이미지 데이터 보강**: 현재 구축된 스키마를 바탕으로 사용자 참여형 장소 이미지 데이터베이스 확장을 목표로 합니다.
